## 2. TDD란? 왜 하는가?
### TDD
- 테스트 주도 개발, Test-driven development
- 테스트를 먼저 작성한 후 개발을 하는 방식의 프로그래밍 방법
#### 장점
- 객체 지향적인 코드를 작성할 수 있다.
- 설계 수정 시간, 디버깅 시간이 단축된다.
- 유지보수하기 용이하다.
- 테스트 문서를 대체할 수 있다.
#### 단점
- 사전 준비 기간이 길다.
- 생산성이 저하된다.
